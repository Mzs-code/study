# 文章总结

## Introducing advanced tool use on the Claude Developer Platform
> 在 Claude 开发者平台上引入高级工具的使用
> https://www.anthropic.com/engineering/advanced-tool-use

文章总结：Claude 高级工具使用 (Advanced Tool Use)
这篇文章来自 Anthropic 工程团队，介绍了三个新的 beta 特性，用于解决 大规模工具调用场景下的效率瓶颈问题。

---
核心痛点

当工具数量多时，工具定义会消耗大量 token（生产环境中可达 5万~13万+ token），每次工具调用都需要单独的模型推理，严重影响性能和成本。

---
三大解决方案

1. Tool Search Tool（工具搜索工具）
- 不再一次性加载所有工具定义，而是让 Claude 按需动态发现相关工具
- 将工具标记为 defer_loading: true，只预加载核心高频工具
- 效果：token 消耗从 ~77K 降到 ~8.7K（减少 85%），准确率显著提升
- 适用场景：10+ 工具、超过 10K token 的多工具/多 MCP 服务器场景

2. Programmatic Tool Calling（编程式工具调用）
- Claude 编写 Python 代码来编排多个工具调用，中间结果不进入上下文窗口
- 效果：token 减少 37%，消除 19+ 次推理轮次，准确率提升
- 举例：处理 20 个团队成员的报销数据，2000+ 行中间数据不会涌入上下文，只返回最终结果
- 适用场景：大数据集处理、多步骤依赖工作流、并行操作、需要显式控制流的条件逻辑

3. Tool Use Examples（工具使用示例）
- 为工具提供具体的使用范例，弥补 JSON Schema 无法传达的使用模式
- 包括：数据格式约定、嵌套结构用法、参数关联模式、可选字段使用时机
- 效果：复杂参数处理场景准确率从 72% 提升到 90%

---
最佳实践

1. 分层策略：先解决最大瓶颈，不要同时上所有特性
2. 优化发现：工具命名和文档要清晰描述，提高搜索匹配效果
3. 文档化返回格式：在工具描述中说明返回数据结构，帮助 Claude 正确解析
4. 精心设计示例：提供 1-5 个覆盖典型和边界情况的示例

---
一句话总结

这篇文章的核心思路是：从静态的全量工具定义，转向智能的动态按需编排，通过"搜索发现 + 代码编排 + 示例引导"三板斧，大幅降低多工具场景下的 token 消耗和延迟，同时提升准确率。
