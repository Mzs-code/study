# Claude Code 内存管理 (Memory)

> 来源: https://code.claude.com/docs/en/memory

## 一、核心概念

Claude Code 有两种跨会话持久化的记忆机制：

| 机制 | 说明 |
|------|------|
| **Auto Memory** | Claude 自动保存的上下文：项目模式、关键命令、用户偏好等 |
| **CLAUDE.md 文件** | 用户手动编写和维护的指令、规则和偏好文件 |

两者在每次会话开始时都会加载到上下文中（Auto Memory 仅加载 `MEMORY.md` 前 200 行）。

---

## 二、内存分层体系

优先级由低到高：

| 层级 | 文件位置 | 用途 | 共享范围 |
|------|----------|------|----------|
| **Managed Policy（组织级）** | `/Library/Application Support/ClaudeCode/CLAUDE.md` | 公司编码标准、安全策略 | 组织内所有用户 |
| **User Memory（用户级）** | `~/.claude/CLAUDE.md` | 个人跨项目偏好 | 仅自己（所有项目） |
| **Project Memory（项目级）** | `./CLAUDE.md` 或 `./.claude/CLAUDE.md` | 项目架构、编码标准 | 团队（通过版本控制） |
| **Project Rules（项目规则）** | `./.claude/rules/*.md` | 模块化主题指令 | 团队（通过版本控制） |
| **Project Local（本地项目级）** | `./CLAUDE.local.md` | 个人项目偏好（自动 gitignore） | 仅自己（当前项目） |
| **Auto Memory（自动记忆）** | `~/.claude/projects/<project>/memory/` | Claude 自动记录的笔记 | 仅自己（每个项目独立） |

**加载规则**：工作目录向上递归查找所有 CLAUDE.md；子目录中的按需加载；更具体的指令优先级更高。

### 层级总览图

```
┌─────────────────────────────────────────────┐
│         Managed Policy (组织级)               │  最低优先级
├─────────────────────────────────────────────┤
│         User Memory (用户级)                  │  ~/.claude/CLAUDE.md
├─────────────────────────────────────────────┤
│         Project Memory (项目级)               │  ./CLAUDE.md
├─────────────────────────────────────────────┤
│         Project Rules (项目规则)              │  ./.claude/rules/*.md
├─────────────────────────────────────────────┤
│         Project Local (本地项目级)            │  ./CLAUDE.local.md
├─────────────────────────────────────────────┤
│         Auto Memory (自动记忆)                │  最高优先级
└─────────────────────────────────────────────┘
```

## 三、Auto Memory 详解

### Claude 会自动记住什么

- **项目模式**：构建命令、测试约定、代码风格
- **调试经验**：疑难问题解决方案、常见错误原因
- **架构笔记**：关键文件、模块关系
- **用户偏好**：沟通风格、工作流习惯、工具选择

### 存储结构

```
~/.claude/projects/<project>/memory/
├── MEMORY.md          # 索引入口，每次会话加载前200行
├── debugging.md       # 主题文件，按需读取
└── ...
```

- `<project>` 基于 git 仓库根目录，同仓库子目录共享内存
- 主题文件不会启动时加载，Claude 需要时按需读取

### 主动要求记住

直接告诉 Claude：`"remember that we use pnpm, not npm"`

### 开关控制

| 方式 | 配置 |
|------|------|
| 交互式 | `/memory` → auto-memory toggle |
| 用户级 | `~/.claude/settings.json` → `"autoMemoryEnabled": false` |
| 项目级 | `.claude/settings.json` → `"autoMemoryEnabled": false` |
| 环境变量（最高优先级） | `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1` |

## 四、/memory 命令使用

在会话中输入 `/memory` 打开内存管理界面：

```
❯ /memory
──────────────────────────────────────────────────────────────────────
 Memory

   Auto-memory: on

   1. User memory                Saved in ~/.claude/CLAUDE.md
   2. Project memory             Saved in ./CLAUDE.md
 ❯ 3. Open auto-memory folder ✔

 Learn more: https://code.claude.com/docs/en/memory
──────────────────────────────────────────────────────────────────────
```

| 选项 | 作用 |
|------|------|
| **Auto-memory: on/off** | 切换自动记忆开关 |
| **User memory** | 编辑 `~/.claude/CLAUDE.md`（个人全局偏好） |
| **Project memory** | 编辑 `./CLAUDE.md`（项目共享指令） |
| **Open auto-memory folder** | 查看/编辑 Claude 自动记录的内容 |

## 五、最佳实践

- **具体明确** — `"Use 2-space indentation"` 优于 `"Format code properly"`
- **结构化组织** — 用 bullet point + markdown 标题分组
- **定期审查** — 随项目演进更新，保持信息准确
- **CLAUDE.md 推荐内容**：常用命令、代码风格约定、项目架构模式
