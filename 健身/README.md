# 健身日记 — 生成规则与偏好

本文件记录 `健身日记.md` 新条目的生成流程、格式规范和我的个人偏好。
**每次帮我生成新条目时，先读本文件，默认按此执行。**

---

## 一、处理流程

1. **我发你的是简要训练内容**：通常包含日期、训练部位、热身、各动作的重量/组数/次数。
   - 内容可能**有缺失**（比如只给了 3 个重量却说 4 组、没写次数、没写组数）。
   - 专业名词可能**描述不准确**（比如「单测」=「单侧」、「斜靠长凳推哑铃」=「上斜哑铃推举」、口语化动作名）。
2. **你帮我调整**：
   - 把口语/简写翻译成标准健身术语（中文名 + 英文名）。
   - 做必要的重量换算（见「换算规则」）。
   - 缺失信息**不要替我编造**：能合理推断的标注清楚（如金字塔递增），不能确定的（如第 4 组的具体重量）在表里如实留白，并在回复里单独问我确认。
3. **写入** `健身日记.md` 的顶部对应月份下、按日期倒序插入（最新的在最上面）。
4. **回复里说明**：列出我替你做的关键处理（术语订正、换算、判定、留白待确认项）。

---

## 二、条目格式

每个条目由三部分组成：

### 1. 标题
```
### 📅 X月X日中午/晚上：<中文主题> (English Theme)
```
- 主题客观描述当天重点，**不要用「轰炸/压榨/巨臂/神级」这类渲染词**。

### 2. 动作表
```
| 动作 (Exercise) | 组数/容量 | 技术要点 |
| :--- | :--- | :--- |
```
- 第一行通常是热身（Warm-up）。
- 动作名：**中文加粗 + 英文**，如 `**史密斯平卧推举**(Smith Flat Press)`。
- 技术要点：开头加粗一个小标签（如 **预疲劳启动**），后接 1-2 句客观要点。

### 3. 训练复盘表
```
#### 📝 训练复盘 (Session Review)
| 维度 (Dimension) | 复盘与建议 (Review & Suggestion) |
| :--- | :--- |
```

---

## 三、复盘规则（重点）

- **去恭维**：不要「恭喜你 / 神级 / 里程碑 / 含金量极高 / 王牌 / 意志力神迹」这类吹捧。客观、就事论事。
  - **但不必矫枉过正**：安排合理就直接说「合理」；没有真问题时「下次微调」一行可省略，不要为凑结构强行挑刺。
- **整体简短**：默认 **3 行左右**。
- **核心点写透，其余各一句**：
  - 挑当天**最核心的 1 个点**（通常是「与上次同部位训练的数据对比 + 追重/调整建议」），可适当写长。
  - 其余维度（动作编排、下次微调等）每条压成**一句话**。
- 建议要**落到具体动作或数字**上，可执行。
- **数据联动（两类都要用）**：
  1. **同部位纵向对比**：翻日记找上一次同部位的重量/容量做对比并给追重建议，优先查下方「六、PB / 进展速查表」拿当前峰值。
  2. **相邻日错峰/恢复**：看前几天练了什么，判断今天的安排是不是合理错峰/恢复（如昨天重背 → 今天练腿）。

## 四、换算规则与固定背景

- **史密斯机/杠铃总重** = 杆重 + 单侧片重 × 2。**史密斯杆按 10kg 计**。
  - 例：单侧 22.5kg → 总重 10 + 22.5×2 = **55kg**。
- 常见术语订正：
  - 「单测」→「单侧」
  - 「斜靠长凳推哑铃」→「上斜哑铃推举 (Incline DB Press)」
  - 口语动作名统一成标准中英术语。
- **固定背景**：身高 178cm、臂围约 40cm。减脂 + 体态修正（对抗久坐含胸/圆肩、胸部内陷）是长期目标。
- **体重（动态，复盘对比一律用最新值）**：当前 **92kg**（2026-06-17）。历史参照：约 94kg（6月初及之前）。
- 符号统一用 `→`，不要用 LaTeX 的 `$\rightarrow$`。

---

## 五、非力量训练条目（有氧 / 拳击 / 体能 MetCon / 康复恢复）

- 这类条目不套用力量日的三段式，**更精简**。
- 动作表可保留，但「组数/容量」列改记**时长 / 距离 / 强度**（如「爬坡 30 分钟」「3 组 × 8 拳组合」）。
- 复盘可**省略**，或只写一句（强度感受，或与相邻力量日的恢复/错峰联动）。
